Épreuve finale

Consignes

  1. Produire les styles CSS de la navigation (principale et secondaire) et de la grille d’items en bas de page.
    • Le HTML est écrit. Vous pouvez le modifier si vous le désirez. Veillez à ce qu’il reste sémantique.
    • Les menus déroulants doivent être fonctionnels.
    • Documentez vos décisions (quel modèle CSS avez-vous choisi, pourquoi ?)
  2. Éditer le HTML générique des deux (2) pages
    • Utilisez toutes les balises sémantiques à votre disposition.
    • Corrigez toute erreur d’accessibilité que vous rencontrez.
    • Veillez à produire un schéma de document logique et lisible.
    • Assurez-vous que votre HTML soit valide.
    • Les menus à ancres devraient être fonctionnels.
    • Insérez les images dans le code.
    • Utilisez les classes CSS mise à votre disposition pour styliser les éléments.
  3. Nommer correctement toutes les pages et ressources (code et image)
  4. Ajustez la communication graphique là où vous la trouvez problématique (rythme, espacements, hiérarchie, dégagement, etc.)
  5. Assurez-vous que le résultat se rapproche le plus possible des maquettes fournies. (dans le dossier « references »)

À considérer

  • Les noms de pages devraient représenter le plus possible ce qu’elles contiennent.
  • Veillez à respecter l’uniformité de la langue dans le vocabulaire.
  • La structure de dossier et URL devraient représenter l’architecture du site le plus possible.
  • Le CSS inclus devrait vous donner des pistes. Consultez-le et récupérez ce qu’il vous faut, là où il vous le faut. Utilisez ses classes à bon escient en produisant votre HTML. Ajoutez des classes là où vous jugez bon de le faire.
  • Documentez votre CSS le plus possible. Documentez celui déjà présent aussi si vous le jugez pertinent.
  • Toutes les balises de marquage de texte (<em>, <strong>, <mark>) ne sont pas mises en place. À vous de le faire.
  • Masquez visuellement les éléments non-désirés mais nécessaires à l’accessibilité.
  • Le HTML de contenu est générique. Éditez-le de façon à ce qu’il soit sémantique. Vous pouvez ajouter/retirer des balises <div> déjà présentes mais ces opérations devraient s’en tenir au minimum. La structure générique actuelle permet le résultat final.
